Decision-safety layer

North star: show exactly where the operator’s judgment changes the outcome — not everything the agents are doing. The Decision Inbox ("Your decisions") is the single union of every card where waitingOn === 'operator', pulled from all lanes — the ONE place to act. The same cards still appear in their pipeline lane, but read-only (a "→ In your decisions" pointer, never a CTA). Coral is reserved for the single most-urgent recommended action; everything else is ink/neutral.

Drawer lifecycle — requested → running (LIVE) → done, in place. Opening a running mission shows the live-follow view: a stage badge (current checkpoint, “no verdict yet”), the latest streamed frame (or an honest awaiting first frame placeholder before any frame arrives), a rolling output tail labeled “recent output (rolling tail) · ~2s”, and the lit checkpoint stepper. On completion the same drawer auto-swaps to the end-report (verdict, outcome, evidence, traces, discussion, the 3 actions) — no reload. The stream is clearly marked demo; a real worker streams stage/output/frames and a verdict is never shown before the run posts one.
Decision clarity layer. The Decision Inbox (“Your decisions”) holds the only human actions; pipeline mirror cards are passive — a quiet “Awaiting your decision · in inbox” label that jumps, never a CTA. Lanes read plainly: Builder tasks (agent shown as Claude / Codex), Runs needing review. Every inbox card carries a what-happens-next line; failed cards carry a failure class; missions carry env/identity badges (environment · identity · data-mode · browser · viewport) and a run contract (target/flow/mode/success/stop/budget/artifacts). The drawer orders evidence forensically — verdict → Expected vs Observed → run timeline → failure frame → replay → diagnostics → raw — and a convert-to-bug preview shows the exact card it would create. States carry a shape glyph as well as color (✕ failed · ▢ blocked · ◐ running · ▲ ready · ✓ done); every control has a visible focus ring. Global controls (Mode: supervised, Pause agents), the digest’s “since you last looked”, dedupe, heartbeat, real checkpoints and forensic data are design-only slots marked preview / awaiting data — never fabricated.

Color profiles — full themes (default: Midnight)

Six switchable profiles. Each defines base · surface · ink · the single act-accent · healthy · degraded · muted. Color = meaning and the accent stays single-purpose in every profile. All derived tokens (softs, lines, text variants, shadows, glow) are computed from these anchors — light profiles darken accent for on-surface text; dark profiles lighten it and add a reserved glow to act + live/active states only. Switching cross-fades surfaces (~0.5s).

Intensity + reduced-motion.A single --mi (0–1) and data-motion="off|low|med|high" on <html> gate the decorative layers: low keeps only slide-ins + the live pulse, med (default, 60%) adds breathing + count pops, high turns on shimmer. @media (prefers-reduced-motion: reduce) hard-disables every animation regardless — fully supported.

Room data model — ports 1:1

Turn = {
  author:  'hermes' | 'claude' | 'codex' | 'test-writer' | 'operator' | 'system',
  target?: same set | 'everyone',          // directed message → arrow in UI
  kind:    'chat' | 'proposal' | 'request_help' | 'status' | 'review',
  time, text, pending?, demo?
}
// Threads group turns by the card/task they're about (thread.cardRef → opens drawer).
// Ranking by kind: request_help / proposal / review = prominent (coral / sage, left border);
//                  chat = normal;  status = dimmed muted telemetry.
Honesty contract — ship this, not the scaffold.The default room renders only real turns. The demoTimeline (synthetic request_help / proposal / typing) exists solely so this prototype's motion can be judged — every entry is demo:true and must be stripped from the production port. Truth-boundary holds: degraded shows ? not 0, empty is honestly empty ("no agent chatter yet"; "no saved suites yet"), a down telemetry source reads ? + "source unavailable" (see Utilities → LLM usage) rather than a fabricated count, and nothing about agent-to-agent dialogue is fabricated at runtime.
